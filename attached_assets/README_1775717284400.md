# IdeaPark

Kompletny starter full-stack dla **IdeaPark**: aplikacja mobilna na **iOS / Android** (Expo React Native), **backend API** (NestJS) i **panel administracyjny WWW** (Next.js).

## Co jest w środku

- `apps/mobile` — aplikacja mobilna premium dla mieszkańca
- `apps/api` — API z logiką rezerwacji, udostępniania miejsc, gości i dostępu
- `apps/admin` — panel administracyjny dla zarządcy / operatora

## Kluczowe funkcje demo

### Mobile
- onboarding i logowanie demo
- dashboard mieszkańca
- moje pojazdy i moje miejsce
- wyszukiwanie wolnych miejsc
- rezerwacja miejsca
- udostępnianie swojego miejsca
- zapraszanie gości
- otwarcie bramy / szlabanu

### API
- logowanie JWT
- seeded demo data
- trwały zapis do pliku JSON
- wyszukiwanie dostępnych miejsc
- walidacja konfliktów rezerwacji
- share windows dla prywatnych miejsc
- guest passes z kodem QR tekstowym
- operacje admina i raporty

### Admin
- dashboard operacyjny
- lista miejsc parkingowych
- lista rezerwacji
- lista gości
- raporty i KPI

## Uruchomienie

### 1. Instalacja zależności

Najpewniejszy wariant:

```bash
cd apps/api && npm install
cd ../admin && npm install
cd ../mobile && npm install
cd ../..
```

Możesz też użyć workspace install z katalogu głównego:

```bash
npm install
```

### 2. Start API

```bash
npm run dev:api
```

API wystartuje na `http://localhost:4000`

### 3. Start panelu admina

W drugim terminalu:

```bash
npm run dev:admin
```

Panel admina będzie dostępny na `http://localhost:3000`

### 4. Start aplikacji mobile

W trzecim terminalu:

```bash
npm run dev:mobile
```

Następnie uruchom:
- `i` dla iOS Simulator
- `a` dla Android Emulator
- albo zeskanuj QR w Expo Go

## Zmienne środowiskowe

### API (`apps/api/.env`)

Skopiuj `apps/api/.env.example` do `.env`:

```bash
cp apps/api/.env.example apps/api/.env
```

### Admin (`apps/admin/.env.local`)

Skopiuj `apps/admin/.env.local.example` do `.env.local`:

```bash
cp apps/admin/.env.local.example apps/admin/.env.local
```

### Mobile

Domyślny adres API jest zapisany w `apps/mobile/src/api/client.ts`.

## Architektura

### Backend
- NestJS
- JWT auth
- file-based persistence JSON
- czyste moduły domenowe
- prosty seed danych demo

### Admin
- Next.js App Router
- client-side fetch do API
- premium dashboard UI
- fallback mock, gdy API nie działa

### Mobile
- Expo / React Native
- piękny custom design system
- szybki tab shell
- fallback mock, gdy API nie działa

## Uwaga produkcyjna

Ta paczka jest **gotowym, uruchamialnym starterem demo / pilot / PoC**, a nie pełnym enterprise deploymentem z PostgreSQL, Redis, CI/CD, billingiem i realnymi integracjami ze szlabanami. Architektura i moduły są jednak przygotowane tak, żeby rozszerzyć je o:
- PostgreSQL / Prisma
- Redis / kolejki
- 2FA
- RODO workflow
- billing SaaS
- realne webhooks i automatykę parkingową
- multi-tenant isolation na poziomie bazy i auth
